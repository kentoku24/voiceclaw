# voiceclaw 要件（暫定）

## 前提 / 方針

- OpenClaw と連携して「アレクサ、◯◯やって」的な体験を LLM ベースで実現する。
- 現時点では **main ブランチへ直接コミット**で進める。
- 要件や設計メモは **docs/ 以下に永続化**していく。

## 必須要件

1. **Wake word で反応するモードに入る**
   - 例: 「ヘイ○○」のような合図で“待受→コマンド受付”へ遷移する。

2. **Wake word の分析は常時起動**
   - 常時マイク入力を監視し、wake word 検出を継続する（低レイテンシ）。

3. **モデル分離（wake word と意図理解/処理）**
   - Wake word 分析: **ローカルモデル**で実行する。
   - Wake word 後の文章（コマンド）理解・処理: **別モデル**で実行する（例: got5.3 など）。

4. **ChatGPT のサブスクリプションで利用可能**
   - ユーザーが ChatGPT サブスクで使える範囲の API/機能で構成する（追加課金や別契約を前提にしない）。

## できれば要件

- **Android 端末を UI とする**
  - 端末側で状態表示（待受/聞き取り中/実行中）や、簡単な操作（開始/停止・ログ確認）を行えると良い。
